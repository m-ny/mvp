# Module 4 Failure Cases

## FC-2026-04-06-01: 通勤包因果诉求与成交状态漏判

### 原始 Transcript（摘要）
- 客户要买日常通勤包。
- 明确说明：每天要带 `15寸电脑+文件`，因此包必须大容量、能装电脑。
- 过程里有负面评价：米白编织托特“有点重”。
- 最终动作：店员推荐轻一点的大号托特，客户“当场买单了”。

### 期望输出
- `visit_purpose` 必须体现因果链，而不是泛场景词：
  - 正确示例：`日常通勤 - 需携带15寸电脑等大容量物品`
  - 错误示例：`日常通勤`
- `purchase_decision_status` 必须为：`已成交`

### 实际失败表现（旧版）
- `visit_purpose` 仅输出：`日常通勤`
- `purchase_decision_status` 输出：`null`

### 复盘结论
- 失败主因 1：名词提取惯性（抓到了“通勤”，漏了“带电脑=>大容量”因果诉求）。
- 失败主因 2：过程负面信息干扰（“嫌重”）压制了最终成交动作。
- 修复策略：仅微调 System Prompt，强化格式与状态判定优先级。
